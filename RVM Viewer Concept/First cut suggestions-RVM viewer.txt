The “3D viewer” tab of the **PCF/GLB Viewer** is implemented in the file `viewer/tabs/viewer3d-tab.js`.  It renders a React component that wires up a complex toolbar (with select/orbit/pan/measure/zoom, fit‑all, orthographic toggle, section box/plane controls), a left‑hand settings panel (heat‑map toggles, overlay scale, support scale, nodes/line‑numbers/length labels, verification mode, section plane padding, etc.), and a right‑hand component/summary panel.  It instantiates a `PcfViewer3D` object (from `viewer/viewer-3d.js`) for all rendering and registers callbacks for uploading PCF, STP or CSV files, adjusting heat‑map buckets, toggling overlay layers, enabling measurement and section tools, and so on.  The `PcfViewer3D` class (≈3 000 lines long) uses Three.js to create geometry, overlays, measurement tools, section box/planes, heat‑maps and overlays; it defines `addComponents()` to load model data (pipes, elbows, flanges, valves, supports, etc.), `buildSupportFallbackComponents()` to draw fallback supports, and numerous methods for navigation/measurement modes, selection, overlays, heat‑map and label rendering.

### 1 Understand the RVM format

* **RVM** is an AVEVA PDMS/Everything3D export format used in plant design and typically paired with a **REV** file containing attributes.  The RVM file is a binary format containing hierarchical model definitions; the REV file is textual and maps element IDs to attribute name/value pairs.  Because there is no Three.js‑native RVM loader, RVM models are usually converted to **glTF** or parsed manually.  Tools such as *aveva-rvm-loader* (npm) or open‑source RVM‑to‑glTF converters can produce geometry and attribute JSON; there is also a Python tool `pdms-rvm-extractor` that writes a glTF plus JSON attributes.  These existing parsers should be used rather than writing a full RVM parser from scratch.  When parsing, remember that PDMS coordinates (X=East, Y=North, Z=Up) must be mapped to Three.js coordinates (X=Right, Y=Up, Z=Back) like the PCF viewer’s `mapCoord` function.

### 3 Implement `RvmViewer3D` (Three.js viewer for RVM)

A skeleton for `RvmViewer3D` can follow the design of `PcfViewer3D` but simplify where possible:

```javascript
// viewer/rvm-viewer-3d.js
import * as THREE from 'three';
import {OrbitControls} from 'three/examples/jsm/controls/OrbitControls.js';
import {CSS2DRenderer, CSS2DObject} from 'three/examples/jsm/renderers/CSS2DRenderer.js';

export default class RvmViewer3D {
  constructor(canvasEl, overlayEl) {
    this.scene = new THREE.Scene();
    this.camera = new THREE.PerspectiveCamera(60, canvasEl.clientWidth / canvasEl.clientHeight, 1, 1e7);
    this.renderer = new THREE.WebGLRenderer({canvas: canvasEl, antialias: true});
    this.labelRenderer = new CSS2DRenderer();
    this.controls = new OrbitControls(this.camera, this.labelRenderer.domElement);
    // initialize grid, axes, helpers, section planes etc.
  }

  // load geometry from the parser
  addComponents(components) {
    components.forEach(comp => {
      // map PDMS coords to Three.js coords
      const mesh = new THREE.Mesh(
         new THREE.BufferGeometry().fromGeometry(comp.geometry),
         new THREE.MeshStandardMaterial({color: comp.color || 0xcccccc})
      );
      mesh.userData = comp.attributes; // store attributes for selection
      this.scene.add(mesh);
    });
    this.fitAll();
    this.render();
  }

  // apply attributes to selection panel
  getAttributesForComponent(component) {
    return component.userData || {};
  }

  setNavigationMode(mode) { /* orbit, pan, measure … */ }
  setSectionMode(mode) { /* box, plane, off … */ }
  loadOverlayLayers(layers) { /* optional overlays from CSV or REV attributes */ }
  fitAll() { this.controls.reset(); /* adjust camera */ }
  render() { this.renderer.render(this.scene, this.camera); this.labelRenderer.render(this.scene, this.camera); }
}
```

This skeleton omits many details—measurement tool, overlays, section clipping, axis gizmos, heat‑maps—that should be ported from `PcfViewer3D` when needed.  Reuse helper functions such as `mapCoord()` and measurement calculations to achieve parity.

### 4 Integrate an RVM parser

* Use an open‑source RVM parser (e.g. `@agilice/aveva-rvm-loader` or `rvm-to-gltf`).  These libraries typically expose an async `parse(buffer)` method returning a scene graph or glTF.  The returned objects include geometry and an ID or name that matches the REV file.
* For each object, build a component record with `geometry` (a Three.js `BufferGeometry`), `attributes` (merged REV attributes), `color`, `id` and `name`.  When merging REV attributes, match the object ID or tag; store attributes in `component.attributes`.  The existing component panel can display these via a generic attributes table.

### 5 Update the attribute and search panels

The current `viewer3d-tab.js` renders attribute rows based on PCF canonical tags.  For RVM models:

* **Attributes panel**: iterate over `component.attributes` keys and values and list them; highlight fields like `ELEMENT_TYPE`, `SPEC`, `NPD`, `PIPE_NAME`, `ZONE`, etc.
* **Search**: implement search across these attributes; reuse the existing search component but index RVM attributes instead of PCF tags.

### 6 Implement “Navis Review (XML tagging)” (Appendix‑A)

* Navisworks’ XML tagging refers to grouping or annotating model items with metadata loaded from XML.  In a React/Three.js context, implement a **Tag Manager**: allow users to import an XML file mapping element IDs to tags.  After parsing, apply tags as `CSS2DObject` labels anchored to the corresponding Three.js mesh (similar to how the PCF viewer overlays CSV labels).
* Provide a left‑panel list of tags; clicking a tag zooms to associated elements.  Use different colors or icons for different tag types.  Reuse functions like `_applyOverlayLayersToViewer` from `viewer3d-tab.js` to attach these tags to the scene.

### 7 Implement “Robust data extraction from RVM” (Appendix‑B)

* Build or integrate a **REV extractor** to parse the REV file into a machine‑readable JSON dictionary.  Map `Element ID` → `{attributeName: value,…}`.  Save this JSON alongside the parsed geometry (e.g. inside the zip deliverable) so other systems can consume it.
* Provide a CLI or Node script (outside the React app) that reads `.rvm` and `.rev`, runs the parser, and outputs glTF plus JSON metadata.  This script can be reused to pre‑convert models before loading them into the viewer.

* **Code structure**: create `viewer/tabs/viewer3d-rvm-tab.js`, `viewer/rvm-viewer-3d.js`, `utils/rvmParser.js` (wrapping the external RVM parser) and update router/navigation to include the “3D RVM Viewer” tab.  Keep consistent naming, styling and state management with existing code.

* **Reasoning**: by copying the 3D viewer tab, we preserve the existing UX and state management.  Encapsulating RVM logic in a separate viewer class avoids polluting PCF logic and makes it easy to evolve.  Using existing RVM‑to‑glTF parsers short‑cuts the complex binary parsing problem.  Mapping PDMS coordinates and attributes ensures the model renders correctly and attributes are searchable.

This plan provides a clear, modular path to deliver a fully functional “3D RVM Viewer” tab that mirrors the current PCF viewer while accommodating the specifics of the RVM/REV format.
